## Secret Manager Investigation

The error shows: 'Secret projects/108005641993/secrets/DATABASE_URL_PRODUCTION/versions/latest was not found'

This means either:
1. The secret DATABASE_URL_PRODUCTION doesn't exist in Secret Manager
2. The secret exists but with a different name (likely lowercase)

You need to either:
A) Create uppercase secrets in Secret Manager, OR  
B) Change deployment to reference existing lowercase secrets

To check what secrets exist:
gcloud secrets list --filter='name:(database OR session)' --format='table(name)'

To create uppercase secrets from existing lowercase ones:
gcloud secrets create DATABASE_URL_PRODUCTION --data-file=<(gcloud secrets versions access latest --secret=database-url)
gcloud secrets create SESSION_SECRET --data-file=<(gcloud secrets versions access latest --secret=session-secret)

OR 

Revert deployment to use lowercase secret names:
DATABASE_URL_PRODUCTION=database-url:latest
SESSION_SECRET=session-secret:latest
